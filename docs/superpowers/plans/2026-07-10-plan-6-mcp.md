# Mass WhatsApp Report — Plan 6: MCP Surface

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Expose the orchestrator to an external brain over MCP — a transport-agnostic core (tools + read-only resource projections + event→notification bridge) with two entrypoints: `stdio` and `streamable-http` (bearer-authed, SSE notifications).

**Architecture:** Thin adapter over Plan 5's composition context — a tool maps to a use-case/`dispatchEngineJob`, a resource maps to a projection read; **no business logic in MCP** (design §2.4). One transport-agnostic `core.js`; `stdio.js`/`streamable-http.js` only bind transports. Domain errors (`code`) map to MCP errors with no stack leakage.

**Tech Stack:** `@modelcontextprotocol/sdk` (**new dependency — absent from the repo**), Express 4 (reuse `apps/api` patterns), `@julio/validation` (yup), `@julio/whatsapp` (domain), Plan 5 `ctx`.

**Depends on:** Plan 5 (composition `ctx`, use-cases, `reconcile`, `buildSnapshot`).

**Source spec:** design §2(3), §9, §11; REQUIREM §2.2 (validation), §4 (auth), §8 (errors).

## Grounding facts (verified — do not re-derive)

- **`@modelcontextprotocol/sdk` is NOT installed anywhere** (0 hits in every `package.json`/`yarn.lock`/`node_modules`). It must be **added** to `@julio/whatsapp-app`. No `StdioServerTransport`/`StreamableHTTP` precedent exists — follow the SDK docs (verify the exact import paths/class names against the installed version at implementation).
- **HTTP:** Express 4 (`apps/api/src/app.js` `createApiApp()`, `apps/api/src/index.js` `app.listen(env.port)`). **Bearer extraction:** `getAuthTokenFromRequest(req)` (`apps/api/src/utils/auth.js`) reads `Authorization: Bearer <token>` (case-insensitive) + cookie fallback. **SSE pattern** (`apps/api/src/controllers/engine-device-events.controller.js`): headers `Content-Type: text/event-stream`, `Cache-Control: no-cache, no-transform`, `Connection: keep-alive`, `X-Accel-Buffering: no`; `res.flushHeaders?.()`; heartbeat `setInterval(()=>res.write(': heartbeat\n\n'), 25_000)`; cleanup on `req.on('close')`; auth checked **before** opening the stream.
- **Config:** `WHATSAPP_MCP_HTTP_PORT: rules.optionalNumber(7300)`, `WHATSAPP_MCP_AUTH_TOKEN: rules.requiredString()` (or `optionalString` + a startup assert) — added in Plan 5's `config/env.js`.
- **Validation:** `@julio/validation` uses **yup** (`^1.4.0`). Author `Yup.object({...}).noUnknown()` schemas and call `await schema.validate(input, { abortEarly:false, stripUnknown:false })` to **reject unknown fields** (§2.2). **No server-side `.validate()`/`.noUnknown()` precedent exists in-repo** — this establishes it. Reuse `flattenValidationErrors` for error messages.
- **Error mapping:** `apps/api/src/utils/response.js` `sendError(res, err)` returns `err.payload` when `err.status && err.payload` else `{ code:'INTERNAL', message:'Internal error' }` — **no stack, no raw message leaked**. Domain `DomainError` (`whatsapp/src/domain/errors.js`) carries `.code`. Map `DomainError.code` → MCP error code+message; everything else → generic internal. Log full `err` via the logger, never serialize it out.
- **Resources / projections:** the domain `reconcile(snapshot)` input shapes ARE the resource projections: `pool={available}`+`config`, `devices=[{queue,onlineAccountIds,...}]`, `campaigns=[{id,status,doneKeys,...}]`. Concrete read pattern: Mongoose `.lean()` + `.select()` + a public-sanitizer map (`getFleetSummary`/`listProxies` in `engine-inventory.controller.js`) — **strip secrets** (`secretRefs`) before returning.
- **Event names** for notifications (`whatsapp/src/domain/events.js`): `account.banned`, `queue.low`, `pool.low`, `campaign.completed`, `report.done`.
